# Design: Bootstrap Payload CMS 3 + PostgreSQL + Testing Infrastructure

## Technical Approach

Run `create-payload-app@latest` with the blank template non-interactively (pipe PostgreSQL as the DB choice). After generation, swap the database adapter from MongoDB to PostgreSQL, install and wire `@payloadcms/plugin-multi-tenant`, scaffold four collections per the architecture design doc §3, and verify the test toolchain (Vitest, Playwright, tsc).

## Architecture Decisions

| Decision | Options Considered | Choice | Rationale |
|----------|-------------------|--------|-----------|
| Scaffold method | `create-payload-app`, manual init, clone starter repo | `create-payload-app --template blank` | Official CLI guarantees correct Next.js admin boilerplate, tsconfig, and vitest config. Manual init risks misconfiguration. |
| Database adapter swap | Interactive CLI selection, post-generation file edits | Post-generation edits | Non-interactive pipe (`printf "PostgreSQL\n"`) is fragile across CLI versions. Accept default, then modify `payload.config.ts` + `package.json` explicitly. |
| Multi-tenant plugin scope | All collections, explicit list | `{ pages: {}, media: {} }` | Users use built-in auth with tenant relationship; Tenants is the source-of-truth (not tenant-scoped). |
| Collection file layout | Flat `src/collections/`, grouped, `src/payload/` | Flat `src/collections/` | Follows Payload 3 convention and blank template structure. |
| Test runner | Vitest (template default), Jest, none | Vitest + `@vitest/coverage-v8` | Already bundled in template; only add coverage provider. |

## Data Flow

```
create-payload-app CLI (blank template, PostgreSQL piped)
  ├─→ agencia-backend/ (full Payload 3 boilerplate)
  │   ├─ payload.config.ts (MongoDB adapter by default)
  │   ├─ src/server.ts (Next.js app entry)
  │   └─ vitest.config.mts, playwright.config.ts, .env.example

Post-generation edits (sdd-apply):
  1. pnpm add @payloadcms/db-postgres @payloadcms/plugin-multi-tenant @vitest/coverage-v8
  2. payload.config.ts: swap mongooseAdapter → postgresAdapter, add multiTenantPlugin
  3. Scaffold src/collections/{Tenants,Users,Pages,Media}.ts per design doc §3
  4. Update .env.example with PAYLOAD_SECRET + DATABASE_URI
  5. Add "typecheck": "tsc --noEmit" to package.json scripts
```

## File Changes

| File | Action | Description |
|------|--------|-------------|
| `agencia-backend/` (entire dir) | Create | Generated by `create-payload-app` blank template |
| `agencia-backend/package.json` | Modify | Swap `@payloadcms/db-mongodb` → `@payloadcms/db-postgres`; add `@payloadcms/plugin-multi-tenant`, `@vitest/coverage-v8`; add `typecheck` script |
| `agencia-backend/payload.config.ts` | Modify | Replace `mongooseAdapter` with `postgresAdapter`; register `multiTenantPlugin` in plugins array |
| `agencia-backend/src/collections/Tenants.ts` | Create | Fields: name, slug, domain, serviceType, frontendType, status, maintenanceFee, projectPrice, paymentStatus |
| `agencia-backend/src/collections/Users.ts` | Create | Extends Payload auth; adds tenant relationship + roles field |
| `agencia-backend/src/collections/Pages.ts` | Create | Fields: tenant (relationship), slug, title, layout (blocks), status, meta (group) |
| `agencia-backend/src/collections/Media.ts` | Create | Fields: tenant (relationship), alt; upload config with mimeTypes + imageSizes |
| `agencia-backend/.env.example` | Modify | Document `PAYLOAD_SECRET` and `DATABASE_URI` with placeholder values |
| `agencia-backend/vitest.config.mts` | Modify | Add `@vitest/coverage-v8` provider config |

## Interfaces / Contracts

Tenant relationship pattern (reused across Pages, Media, Users):

```typescript
// Standard field in tenant-scoped collections
{ name: 'tenant', type: 'relationship', relationTo: 'tenants', required: true, index: true }
```

Multi-tenant plugin wiring (the key non-obvious pattern):

```typescript
// payload.config.ts
multiTenantPlugin({
  collections: { pages: {}, media: {} },
  tenantField: 'tenant',
  tenantsArrayField: 'tenants',
  tenantsSlug: 'tenants',
  userHasAccessToAllTenants: (user) => user.roles?.includes('super-admin'),
})
```

## Testing Strategy

| Layer | What to Test | Approach |
|-------|-------------|----------|
| Unit | Collection field definitions, access control callbacks | Vitest — test exported config objects and ACL functions |
| Integration | Plugin wiring, database connectivity, tenant isolation | Vitest — spin up Payload with test DB, verify collection registration and tenant filtering |
| E2E | Admin UI loads, login flow, tenant-scoped data visibility | Playwright — template provides admin smoke tests; extend with tenant isolation assertions |
| Type | All source files | `tsc --noEmit` via `typecheck` script — zero errors required |

## Migration / Rollout

No migration required. Greenfield scaffold — `agencia-backend/` directory does not yet exist. Rollback: delete directory and drop the PostgreSQL database.
